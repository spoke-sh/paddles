 1 # Model-Directed Routing Proof
 2 
 3 ## Summary
 4 
 5 This voyage replaced the primary top-level controller classifier in the
 6 `MechSuitService` runtime with model-directed first action selection.
 7 
 8 ## Before
 9 
10 The application runtime used a controller-side shortcut before the planner lane:
11 
12 - `casual` turns stayed on the synthesizer lane immediately
13 - other turns were heuristically mapped into direct response, deterministic
14   action, or planner work before the planner model saw interpretation context
15 
16 That meant `AGENTS.md` and linked foundational docs could influence later
17 planning and synthesis prompts, but not the first route decision.
18 
19 ## After
20 
21 The primary runtime path now behaves as:
22 
23 1. load `AGENTS.md` and linked foundational guidance
24 2. assemble interpretation context
25 3. ask the planner lane for the first bounded action
26 4. validate and execute the selected route
27 5. recurse or synthesize from the resulting evidence
28 
29 Implemented in:
30 
31 - `src/application/mod.rs`
32 - `src/domain/ports/planning.rs`
33 - `src/infrastructure/adapters/sift_agent.rs`
34 - `src/infrastructure/adapters/sift_planner.rs`
35 
36 ## Concrete Turn Proof
37 
38 The regression test `application::tests::process_prompt_assembles_interpretation_before_model_selected_initial_action`
39 proves the runtime no longer commits the route before planner selection.
40 
41 It exercises a turn that previously would have been trapped behind the cheap
42 `casual` shortcut:
43 
44 - prompt: `Howdy`
45 - interpretation context: loaded from workspace `AGENTS.md`
46 - planner-selected first action: `answer`
47 - resulting classified intent: `direct-response`
48 
49 The test asserts:
50 
51 - interpretation context is assembled before `PlannerActionSelected`
52 - `PlannerActionSelected` occurs before `IntentClassified`
53 - the planner request contains `AGENTS.md`-derived interpretation sources
54 - the final route is a model-selected direct response, not a pre-planner
55   controller shortcut
56 
57 ## Supporting Regression Coverage
58 
59 - `application::tests::answer_initial_actions_route_to_direct_responses`
60 - `application::tests::tool_initial_actions_route_to_deterministic_execution`
61 - `application::tests::resource_initial_actions_route_to_the_planner_loop`
62 - `application::tests::stop_initial_actions_fall_back_to_direct_responses`
63 - `infrastructure::adapters::sift_agent::tests::initial_action_prompts_include_interpretation_context`
64 
65 ## Remaining Transitional Debt
66 
67 The backbone contract is delivered for the main `MechSuitService` runtime.
68 Remaining debt is narrower:
69 
70 - the `tool` first action is still a bridge into the older deterministic tool
71   runtime
72 - legacy direct adapter helper methods outside the main service path still carry
73   heuristic intent inference
74
