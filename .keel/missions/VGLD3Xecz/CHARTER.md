# Generalize Paddles As A Capability-Negotiated Recursive Harness - Charter

Archetype: Strategic

## Goals

| ID | Description | Verification |
|----|-------------|--------------|
| MG-01 | Deliver an active epic slice that refactors Paddles around stable session, capability, and execution-hand interfaces so the recursive context harness generalizes across model providers without losing its local-first, visible, evidence-backed turn loop. | board: VGLD4Iesy |

## Constraints

- Preserve the recursive context harness as the product essence. The planner/controller loop, operator-visible traces, and evidence-backed synthesis path must remain first-class.
- Keep the rollout local-first. Do not require hosted harness orchestration, IDE-fed context, external broker dependencies, or remote secret-bearing sandboxes.
- Favor stable interfaces over provider-specific branching. Session durability, capability negotiation, execution hands, and profile semantics should be repo-owned contracts rather than hard-coded per-provider assumptions.
- Keep security structural where possible. Credentials and privileged transport state should move farther away from generated code and shell execution, not rely only on prompt discipline.
- Maintain traceability. Any new session, hand, capability, or profile layer must remain visible through existing recorder, manifold, forensic, transit, and diagnostics surfaces.
- Update the owning docs and board artifacts when behavior changes. Architecture, configuration, and planning contracts must stay aligned with the implementation.

## Halting Rules

- DO NOT halt while epic `VGLD4Iesy` has any draft, planned, active, or verification-pending voyage/story work.
- HALT when epic `VGLD4Iesy` is complete and the runtime is backed by board-linked evidence for durable session semantics, capability-negotiated harness behavior, decoupled execution hands, and adaptive harness profiles.
- YIELD to the human if the remaining decision requires product direction on which profiles or hands should be exposed by default, or whether any new capability should become operator-facing immediately.
